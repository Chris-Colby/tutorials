How to Host a Static Website on Amazon Web Services - AWS S3

  1. Excellent Resources - Open these and reference them if you wish. They really helped me!
    How to Host your Static Website with AWS - A Beginner's Guide
    Hosting your static site with AWS S3, Route 53, and CloudFront
  2. Open this, too! It's a good idea to work alongside the Amazon Documents in the event of updates. THE OFFICIAL STEPS FROM AMAZON

  3. LET'S GET STARTED!

  4. Go to AWS (https://aws.amazon.com) - Sign In to the Console
  5. Click Services
  6. Click S3
  7. Click Create Bucket
  8. In “Bucket name” text field - Name the Bucket - NOTE: (no spaces, only valid characters - no ".com" necessary) - AWS provides a link underneath for “Rules for bucket naming” - Do not include dots in the name!
  9. AWS Region - Choose Region - NOTE: for me, this is “US East (N. Virginia) us-east-1”. The bucket's Region may be pre-selected above Bucket Name text field....mine was the most recent time I did this.
  10. Scroll to bottom - Click Create Bucket
  11. Click on the newly created bucket
  12. See tabs across the top
  13. Next, upload my website files to my new bucket by clicking Upload button
  14. NOTEI choose to drag and drop. I did not move JOJ master folder over, rather, open the folder on my desktop, select and drag all the folders and files from within my Master Folder and drag them into the AWS upload box. Ex. go to my JOJ master folder, open it and select all the individual files and folder inside the joj master folder and drag all the individual folders and files into S3 Upload area.
  15. Click the Upload button
  16. After upload completes - get green Upload Succeeded!
  17. Single click on index.html file - Now taken to a new tab - will see owner, last modified, size, key. Can also - if lots of files and folders to fit on one page - maybe this is better: under “Files and folders” click on the file under the Name tab - - in search field “Find by name” - can input either index or the page I'd like to see.
  18. NOTE: the OBJECT URL (inside the properties tag in the Object Overview box) - this represents the file inside AWS - and can be used for public access - but, just because AWS creates this file usable for public access does not mean that the html file / this link is accessible
  19. TRY opening the Object URL link in another tab - will get error - XML file does not appear to have any style info. Code Access Denied, Message Access Denied, etc. This means that the file is not made public (yet) - I don not have the proper public permissions yet
  20. Go back to Object Overview tab - In the same index.html overview tab, click the Open button. Now I can see the file contents. NOTE: the website won’t show up as finished product - only the html shows up - the skeleton - latest-my photos didn't show up. Example, click any link and I get the XML error / Access Denied message. Simply uploading all of our files does not mean our website is going to work. We need to complete more steps. In the above steps, I created my bucket and uploaded my files.

  21. Next Part - configure my bucket to host my static website.

  22. Navigate to AWS S3 - Top Left: Click Services (make sure I'm still logged in)
  23. Click S3
  24. Click on my bucket
  25. Select PROPERTIES tab
  26. Scroll down - In STATIC WEBSITE HOSTING area Click Edit button - This is the configuration I need to set to enable my bucket to host my website
  27. Under “Static website hosting” Select ENABLE (Radio Button) / more info appears
  28. Select - Enable - under static website hosting
  29. More options present themselves — go to index document text box and type in index.html - to denote that this is the document I want displayed when someone types in my url - this is my main homepage
  30. Optional - enter in the error.html document - the document that gets displayed when a user tries to access pages that are not available - can make this page really cool - either way, enter an html document here - with directions and a link back to the homepage. Note - this is the page with the redirection rules and notes. Optional, but considerate. It's good form.
  31. Click Save Changes button - I got a green note and check mark - “Successfully edited static website hosting” message in green bar up top
  32. Doublecheck that the static website hosting is enabled - scroll down on the properties tab/page - it should have some info in it now - remeber to copy the http link created for my site
  33. Above, we generated an endpoint URL - and configured my error page NOTE: my endpoint url - it looks something like this: http://MY-BUCKET-NAME.s3-website-us-east-1.amazonaws.com Note - when I tried to open this, I got the 403 Forbidden message. Access denied - - This is normal, see below…

  34. Next Part - configure security and Permissions - to make my site public for users on the internet

  35. Navigate back to AWS S3 - Top Left: Click Services (make sure I'm still logged in)
  36. Click S3
  37. Click on my bucket
  38. NOTE: by default s3 objects are not made public even when bucket hosting is enabled. Even if an endpoint is configured, still no go!!! - trying it gives us an hsttp 403 error - access denied - go ahead - paste my endpoint url and will get 403 forbidden. To remedy this, need to perform 2 actions in my bucket: 1 - Need to unblock public access to my bucket, AND 2 - make my website files public
  39. Goto AWS console - click services - Storage-S3
  40. Click on the bucket I want to open configuration tabs
  41. Click on PERMISSIONS TAB
  42. Click on EDIT under Block public access (bucket settings)
  43. UNCHECK box “Block all public access”
  44. Doublecheck that all other checkboxes in this section are unchecked
  45. Click SAVE CHANGES button - Caution box appears
  46. Type in “confirm” in the caution box
  47. Click Confirm Button
  48. Click on Objects tab
  49. Click the REFRESH ICON - the circle with the arrow pointer right/clockwise

  50. DO THE EDIT BUCKET POLICY NOW

  51. Select Permissions tab
  52. Click Edit Button in Bucket policy section
  53. Paste THIS code in the text field under “Policy:”
  54. {
                              "Version":"2012-10-17",
                              "Statement":[
                                {
                                  "Sid":"PublicReadGetObject",
                                  "Effect":"Allow",
                                  "Principal": "*",
                                  "Action":["s3:GetObject"],
                                  "Resource":["arn:aws:s3:::your-bucket-name/*"]
                                }
                              ]
                            }
  55. Above - where it says “your-bucket-name” = put in your bucket name!!!!! And Keep the /*
  56. Click Save changes button
  57. OK - go try the site
  58. Services > S3 > Select Bucket > Click refresh button > Properties tab > scroll to Static website hosting > click the link - opens in a new tab
  59. Check your work!
  60. Start debugging!
